# 🍉 GUA Token｜瓜币（贡献积分 → 贡献凭证 → 治理工具）

> **GUA 不是“炒作代币”。**  
> 它是「吃瓜群众自治社」内部的 **贡献与治理体系**：把参与变成可验证的贡献记录（Proof-of-Contribution），并逐步演进出可用的投票权与权益体系。  
> **本仓库只聚焦：GUA（瓜币/瓜积分）这个项目本身**。

---

## 这是什么？（TL;DR）

**GUA（瓜币 / 瓜积分）** 是社区内部的贡献计量单位，代表：

- ✅ 参与度（Participation）
- ✅ 贡献力（Contribution）
- ✅ 治理权（Governance Power）

它的目标用途（设计方向）：

- 🗳️ **投票权**：决定选题、内容走向、奖励规则、提案表决  
- 🎁 **贡献奖励**：对内容创作、互动、推广、协作交付进行奖励  
- 🎟️ **专属权益**：兑换身份/徽章、活动资格、allowlist、门票等  
- 🧩 **治理参与**：让“贡献者”拥有更高的参与权，而不是只看嗓门

> **核心原则：先把规则跑通，再考虑上链。**  
> 先让“贡献可被验证、可被复核、可被奖励”，再讨论 Token 化与链上治理。

## 范围与边界（Scope & Non-Goals）

### 我们要做的（Scope）
- 定义 **GUA 积分如何获得**：任务、贡献、协作、审核等
- 定义 **贡献如何被验证**：证据、审计、复核机制（PoC）
- 定义 **如何用 GUA 参与治理**：投票范围、提案流程、权重规则
- 定义 **权益体系**：Discord 角色/徽章、allowlist、活动资格等
- 定义 **反作弊与风控**：防刷分、防冒领、防女巫攻击（最小可行）

### 我们不做的（Non-Goals）
- 不承诺价格、不讨论“涨跌预测”
- 不以“发币”为目的推进项目
- 不要求任何人提供私钥/助记词/转账
- 不做任何带金融承诺的营销行为

## 机制概览（v0.1）

### 1) GUA 积分（链下）
- **来源**：点赞评论、内容任务、有效选题、协作交付、质检、运营等  
- **记录**：可审计（建议统一在 GitHub/Notion/Sheet 中留证）  
- **结算**：按周期结算（周/月），避免“即时刷分”

### 2) Proof-of-Contribution（贡献证明）
建议优先采用 **可验证证据**：
- GitHub：PR / Issue / Review / 文档提交
- 内容：脚本、数据图表、校验报告、投票记录
- 运营：活动策划、任务执行、复盘总结（需可被复核）

### 3) 治理（轻量版）
- 先从链下投票开始（Notion/表单等）  
- 投票范围：选题、规则更新、奖励调整、重大合作等  
- 随社区规模升级到更透明工具（是否上 Snapshot 等，由社区决定）

### 4) Token 化路线（后置、可选）
上链不是目的，是工具。上链前必须明确：
- 发行/分配口径与边界
- 风控规则与反作弊方案
- 治理工具与权限模型
- 法规与合规风险（至少做到风险提示与边界声明）

## 如何参与（How to Contribute）

你不需要懂合约也能参与。最推荐从这几类开始：

- **写文档**：把规则写清楚（最缺）
- **做模板**：提案模板 / 贡献证明模板 / 复盘模板
- **做风控**：反作弊、反女巫、申诉与仲裁流程
- **做工具**：积分台账模板、自动化统计脚本（可选）

贡献流程：
1. 去 Issues 找 `good first issue` / `help wanted`
2. 认领后提交 PR（文档/模板/规则都算）
3. 合并后记入贡献记录（PoC）
